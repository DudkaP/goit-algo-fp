# Final Project — goit-algo-fp

## Запуск

1. Створити та активувати venv
2. Встановити залежності:

- `pip install -r requirements.txt`

3. Запуск завдань:

- `python task_1_linked_list.py`
- `python task_2_pythagoras_tree.py`
- `python task_3_dijkstra_heap.py`
- `python task_4_visualize_heap.py`
- `python task_5_tree_traversals_visual.py`
- `python task_6_greedy_vs_dp.py`
- `python task_7_monte_carlo_dice.py`

## Завдання 1 — Однозв’язний список

Реалізовано:

- реверсування списку зі зміною посилань між вузлами;
- сортування однозв’язного списку методом злиття (merge sort);
- об’єднання двох відсортованих списків в один відсортований.

## Завдання 2 — Фрактал “дерево Піфагора”

Фрактал будується рекурсивно. Користувач вводить рівень рекурсії, після чого програма візуалізує дерево.

## Завдання 3 — Алгоритм Дейкстри з бінарною купою

Реалізовано пошук найкоротших шляхів у зваженому графі від стартової вершини до всіх інших з використанням `heapq` для оптимізації вибору вершини.

## Завдання 4 — Візуалізація бінарної купи

З масиву (купі) будується дерево, де для індексу `i` діти мають індекси `2i+1` та `2i+2`. Далі дерево візуалізується через `networkx`.

## Завдання 5 — Візуалізація обходів DFS та BFS

Обхід виконується без рекурсії:

- DFS — через стек
- BFS — через чергу
  На кожному кроці вузол отримує унікальний колір (градієнт від темного до світлого), що відображає порядок відвідування.

## Завдання 6 — Жадібний алгоритм та динамічне програмування

Реалізовано:

- жадібний вибір страв за максимальним співвідношенням `calories/cost`;
- оптимальний вибір через DP (0/1 knapsack) при заданому бюджеті.
  DP гарантує оптимум, greedy може бути неоптимальним.

## Завдання 7 — Метод Монте-Карло (2 кубики)

Змодельовано велику кількість кидків двох кубиків, отримано емпіричні імовірності сум 2..12.
Результати порівняно з аналітичними імовірностями (кількість способів / 36).
Зі збільшенням кількості симуляцій похибка зменшується.
